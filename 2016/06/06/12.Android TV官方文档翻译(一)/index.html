创建一个TV工程

该部分讨论了如何修改一个现有的应用或者创建一个新应用来运行在TV设备上。在TV设备上运行的App必须有以下部分。

  • Activity for TV(必须)-在manifest中，声明activity是运行在TV设备中.
  • TV 依赖库(可选)-有一些可供电视设备开发的依赖库，提供一些用户交互的控件。

前提

在开始编译一个电视apps之前，必须：

  • 升级SDK tools版本到24.0.0或者以上版本
    升级的SDK tools能使你在电视上编译或者测试App
  • 升级SDK到Android 5.0(API 21)或者以上版本
    升级的SDK版本能够为TV app提供新的API
  • 创建或者升级你的app工程
    为了电视设备能使用新的API，你必须创建一个或者将现有的工程SDK升级到Android 5.0(API level 21) 或者更高版本

声明一个Activity

一个运行在电视上的app必须在manifest文件中必须用CATEGORY_LEANBACK_LAUNCHER的intent filter 声明一个launcher activity. 这就说明你的应用用于TV，若要放在Google Play上也必须有这样的要求。声明这个intent也说明了当用户在电视主界面点击你应用图标的时候会启动这个页面.

声明Leanback support

声明为Android TV所用的Leanback用户交互.如果应用不仅运行在手机同样运行在Android TV，那么将required属性的值设置为false.如果required属性的值设置为true,那么设备仅仅会使用Leanback UI.

声明touchscreen为不必须

添加TV支持的依赖库

运行在TV上的app的SDK应该包含支持的依赖库，这些库提供了在电视端用户交互的API，这些库位于<sdk>/extras/android/support/目录下。下面是库的列表：

  • v17 leanback library - 提供了用户交互的控件，特别是媒体播放
  • v7 recyclerview library - 提供了一些高效的内存管理来显示lists,一些在v17 leanback库中的类依赖于这个库。
  • v7 cardview library - 提供了展示信息卡片的用户交互控件，像媒体每项的图片和描述。

如果你决定使用v17 leanback库，你应该说明其依赖于v4 support library.这就意味着使用leankback 依赖库包含这些库:

  • v4 support library
  • v7 recyclerview support library
  • v17 leanback support library

v17 leanback库包含的资源，需要你用具体的步骤，包括在应用项目中。
关于导入依赖库的相关说明，可以参考Support Library Setup

运行TV Apps

创建虚拟TV终端

  1. 启动AVD Manager.
  2. 在AVD Manager的对话框中点击Device Definitions项
  3. 选择电视端定义的一个并且点击Create AVD
  4. 选择模拟器选项并且点击OK来创建AVD
