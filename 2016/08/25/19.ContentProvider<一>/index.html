ContentProvider<一>

Posted on 2016-08-25   |   In Android   |  

ContentProvider实现跨包的数据共享。standard_prefix:/authority/data_path/id

standard_prefix:始终是content://
authority:内容提供者的名称
data_path：请求数据类型
id:指定请求的特定记录

读取通信录中联系人信息

在这里遇到一个权限问题

在Android 6.0以下，没有运行时权限一说，要读取通信录中的数据在AndroidMainifest.xml中声明权限就可以了。
uses-permission android:name=”android.permission.READ_CONTACTS”

在6.0以上涉及到运行时权限，

###managedQuery()关于投影，筛选，排序

这个在6.0以上已经过期
具体见如下示例

```
/**

 * projection :投影 控制查询返回的序列，上面projection返回 id,名字和手机号
 * 参数3，4用于筛选
 * 最后一个用于排序
 */

if (Build.VERSION.SDK_INT < 11){
    testCursor  = managedQuery(allContacts,projection, ContactsContract.Contacts.DISPLAY_NAME 
            + " LIKE ?",new String[]{"%Lee"},ContactsContract.Contacts.DISPLAY_NAME+"AES" );
}else{
    CursorLoader cursorLoader = new CursorLoader(this,allContacts,projection,ContactsContract.Contacts.DISPLAY_NAME
            + " LIKE ?", new String[]{"%Lee"} , ContactsContract.Contacts.DISPLAY_NAME+"AES");
    testCursor =  cursorLoader.loadInBackground();
}
